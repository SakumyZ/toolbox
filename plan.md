## 新增功能规划

> 当前统计：已完成 4 项，部分完成 1 项，未开始 5 项

### 一、系统工具类

#### 1. Windows 服务管理器 ⭐ 优先级：高

> 用户明确需求

当前状态：🟡 部分完成（已支持列表、搜索筛选、收藏、单项启停重启、分组批量启停；未支持服务描述展示、启动类型修改、实时状态轮询/WMI 监听）

**功能：**

- 列出所有 Windows 服务（名称、状态、启动类型、描述）
- 启动 / 停止 / 重启服务
- 修改启动类型（自动 / 手动 / 禁用）
- 搜索 + 筛选（按状态、按名称关键字）
- 常用服务置顶/收藏（如 MySQL、Redis、IIS、Docker Desktop Service）
- 一键批量启停（场景化：如"启动开发环境"一键启动 MySQL + Redis + Nginx）

**技术要点：**

- `System.ServiceProcess.ServiceController` API
- 需要管理员权限（启停操作时提权）
- 实时状态轮询或 WMI 事件监听

---

#### 2. 端口占用查看器 ⭐ 优先级：高

> 开发常见痛点：端口被占用不知道是哪个进程

当前状态：✅ 已完成（当前规划内核心能力已具备）

**功能：**

- 列出所有监听中的 TCP/UDP 端口
- 显示占用进程（PID、进程名、路径）
- 搜索特定端口号
- 一键结束占用进程
- 常用端口高亮标记（80, 443, 3000, 3306, 5432, 6379, 8080...）

**技术要点：**

- `netstat` / `GetExtendedTcpTable` Win32 API
- `Process.GetProcessById` 获取进程信息

---

#### 3. Hosts 文件编辑器 ⭐ 优先级：中

> 开发经常需要切换 hosts 配置

当前状态：⏳ 未开始

**功能：**

- 可视化编辑 `C:\Windows\System32\drivers\etc\hosts`
- 单行启用/禁用（注释切换）
- 分组管理（如：开发环境、测试环境、生产环境）
- 方案切换（一键切换整组 hosts 配置）
- 语法校验 + 重复检测

**技术要点：**

- 文件读写需要管理员权限
- 备份/恢复机制

---

#### 4. 环境变量管理器 ⭐ 优先级：中

> 比系统自带的环境变量编辑器好用 100 倍

当前状态：⏳ 未开始

**功能：**

- 分区显示：用户变量 / 系统变量
- PATH 变量特殊处理：拆分为列表，支持拖拽排序
- 搜索变量名/值
- 新增 / 编辑 / 删除变量
- 导出/导入配置（方便迁移电脑）
- 变量值路径有效性检测（标红失效路径）

**技术要点：**

- `Environment.GetEnvironmentVariable` + 注册表读写
- 修改系统变量需要管理员权限
- 广播 `WM_SETTINGCHANGE` 使修改立即生效

### 二、开发工具类

#### 5. JSON / 数据格式化工具 ⭐ 优先级：高

> 每天都在用的工具，不用再开网页了

当前状态：⏳ 未开始

**功能：**

- JSON 格式化 / 压缩 / 校验
- JSON ↔ C# Class 互转
- JSON Path 查询
- Base64 编码/解码
- URL 编码/解码
- 时间戳 ↔ 日期时间 转换
- UUID/GUID 生成
- Hash 计算（MD5、SHA1、SHA256）

**技术要点：**

- `System.Text.Json` 处理 JSON
- 纯本地计算，无需网络

---

#### 6. 正则表达式测试器 ⭐ 优先级：中

当前状态：⏳ 未开始

**功能：**

- 输入正则 + 测试文本，实时高亮匹配结果
- 显示捕获组
- 常用正则模板库（邮箱、手机号、IP、URL 等）
- 正则语法解释

**技术要点：**

- `System.Text.RegularExpressions.Regex`
- 实时匹配 + 超时保护（防止 ReDoS）

---

#### 7. SSH Config 预设切换 ⭐ 优先级：高

> 在公司 / 个人等多套 SSH config 之间快速切换，避免手动注释

当前状态：✅ 已完成（首版）

**功能：**

- 保存多套 SSH config 预设
- 支持新增 / 编辑 / 删除预设
- 支持从当前 `C:\Users\housei\.ssh\config` 导入为预设
- 点击预设后直接覆盖当前 ssh config
- 自动备份当前 config，避免误覆盖
- 显示当前生效预设与预设内容预览
- 后续预留规则级控制扩展位

**技术要点：**

- SQLite 持久化预设元数据与整份 config 文本
- 文件写入前进行基础校验
- 切换时自动备份 `config.backup.yyyymmddHHmmss`
- 首版采用“预设覆盖式”，暂不做逐 Host 规则级编辑

---

#### 8. Skill 切换管理 ⭐ 优先级：高

> 用于控制 agent 可见的 skill 范围，减少误用与串用

当前状态：✅ 已完成（首版）

**功能：**

- 扫描启用目录中的全部 skill 文件夹并展示
- 扫描停用目录中的全部 skill 文件夹并展示
- 使用 `ToggleSwitch` 一键启用 / 停用 skill
- 停用时将 skill 文件夹移动到停用目录，启用时移动回启用目录
- 支持配置启用目录与停用目录（默认 `~\.agents\skills` 与 `D:\Backup\AI\skills`）
- 支持为 skill 设置别名，列表优先展示别名
- 支持为 skill 设置分类，便于筛选和观察
- 支持按名称 / 别名 / 描述搜索，按分类与状态筛选
- 自动读取 `SKILL.md` 描述作为列表说明

**技术要点：**

- 文件夹级目录扫描与移动实现启停切换
- SQLite 持久化目录配置、skill 别名与分类信息
- 页面内进行状态筛选、分类筛选与搜索过滤
- 目标目录冲突保护，避免覆盖已有 skill 文件夹

### 三、效率工具类

#### 9. 定时器管理 ⭐ 优先级：中

> 管理各种提醒频率，应用运行时弹出 Windows 默认通知

当前状态：✅ 已完成（首版）

**功能：**

- 提醒列表管理（新增 / 编辑 / 删除 / 启用 / 停用）
- 提醒频率支持单次、间隔分钟、每天、每月
- 提醒分类与预设模板（如喝水、起身、会议、下班）
- 自定义提醒文案、提醒标题、通知时间
- 默认单次提醒，触发完成后自动停用
- 到点弹出 Windows 默认 notification
- 显示下一次触发时间与最近一次触发记录

**技术要点：**

- SQLite 持久化提醒配置与触发记录
- `DispatcherTimer` 或 `PeriodicTimer` 进行应用运行时调度
- Windows App SDK 通知能力（默认系统通知）
- 调度服务与通知服务解耦，便于后续扩展托盘常驻、稍后提醒、工作日规则
- 首版约束：应用关闭后不继续提醒

**本轮实装清单：**

- [x] 新增“定时器管理”导航入口与管理页面
- [x] 新增提醒配置表与触发记录表
- [x] 支持新增 / 编辑 / 删除 / 启用 / 停用提醒
- [x] 支持单次、间隔分钟、每天、每月四种提醒频率
- [x] 支持提醒分类预设（喝水、起身、会议、下班、自定义）
- [x] 支持显示下一次触发时间与最近一次触发时间
- [x] 应用运行时自动轮询并到点弹出 Windows 默认通知
- [ ] Cron 表达式高级设置（后续）
